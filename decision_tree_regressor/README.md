# Дерево решений — Регрессор

## Теоретические основы

### Что такое дерево решений?

**Дерево решений (Decision Tree)** — это непараметрический метод машинного обучения, используемый для задач классификации и регрессии. В случае **регрессии** модель предсказывает непрерывное числовое значение.

Дерево решений строит модель в виде иерархической древовидной структуры, состоящей из следующих элементов:

| Элемент структуры | Описание |
|---|---|
| **Корневой узел (Root Node)** | Начальная точка дерева, содержащая весь набор данных |
| **Внутренние узлы (Internal Nodes)** | Узлы, в которых происходит разбиение данных по определённому признаку и порогу |
| **Ветви (Branches)** | Соединяют узлы и представляют условия разбиения |
| **Листовые узлы (Leaf Nodes)** | Конечные узлы, содержащие предсказанное значение — среднее по объектам обучающей выборки, попавшим в данный лист |

### Принцип рекурсивного бинарного разбиения

Алгоритм работает по принципу **рекурсивного бинарного разбиения (Recursive Binary Splitting)**:

1. На каждом шаге перебираются все признаки и все возможные пороговые значения. Выбирается та комбинация (признак, порог), которая минимизирует функцию потерь (например, MSE).

2. Данные разделяются на две группы:
   - **Левая ветвь:** объекты, у которых значение признака $\leq$ порога.
   - **Правая ветвь:** объекты, у которых значение признака $>$ порога.

3. Для каждой из двух групп процесс повторяется рекурсивно.

4. Разбиение прекращается при выполнении условий остановки (максимальная глубина, минимальное число объектов в узле и т.д.).

5. **Предсказание** для нового объекта — это среднее значение целевой переменной объектов обучающей выборки, попавших в соответствующий лист.

### Интуитивный пример

Допустим, мы предсказываем **цену квартиры**:

```
Площадь > 50 м²?
├── Да → Этаж > 5?
│   ├── Да → Предсказание: 8 000 000 руб.
│   └── Нет → Предсказание: 6 500 000 руб.
└── Нет → Количество комнат > 1?
    ├── Да → Предсказание: 4 500 000 руб.
    └── Нет → Предсказание: 3 000 000 руб.
```

В данном примере модель последовательно разбивает пространство признаков на регионы и в каждом листе возвращает среднюю цену квартир обучающей выборки, попавших в этот регион.

## Математический аппарат

### Критерии разбиения (Splitting Criteria)

#### MSE — Mean Squared Error (по умолчанию в sklearn)

Для узла $m$ с набором объектов $Q_m$:

$$MSE(Q_m) = \frac{1}{|Q_m|}\sum_{i \in Q_m}(y_i - \bar{y}_m)^2$$

где $\bar{y}_m = \frac{1}{|Q_m|} \sum_{i \in Q_m} y_i$ — среднее значение целевой переменной в узле.

При разбиении узла $m$ на подмножества $Q_{left}$ и $Q_{right}$ выбирается такое разбиение, которое минимизирует взвешенную сумму MSE дочерних узлов:

$$Loss = \frac{|Q_{left}|}{|Q_m|} \cdot MSE(Q_{left}) + \frac{|Q_{right}|}{|Q_m|} \cdot MSE(Q_{right})$$

Таким образом, алгоритм жадно выбирает на каждом шаге ту пару (признак, порог), которая максимально снижает суммарную ошибку в дочерних узлах.

#### MAE — Mean Absolute Error

$$MAE(Q_m) = \frac{1}{|Q_m|}\sum_{i \in Q_m}|y_i - median(y_m)|$$

При использовании MAE предсказание в листе — это **медиана**, а не среднее. MAE является более устойчивым к выбросам критерием, поскольку абсолютные отклонения не усиливают влияние аномальных значений в отличие от квадратичных.

## Гиперпараметры

| Параметр | Описание | По умолчанию | Влияние |
|---|---|---|---|
| `max_depth` | Максимальная глубина дерева | `None` (без ограничений) | Больше глубина — больше переобучение; основной параметр контроля сложности |
| `min_samples_split` | Минимальное число объектов для разбиения узла | `2` | Больше значение — меньше переобучение |
| `min_samples_leaf` | Минимальное число объектов в листовом узле | `1` | Больше значение — более гладкие предсказания |
| `max_features` | Число признаков, рассматриваемых при выборе разбиения | `None` (все признаки) | Меньше значение — больше случайность, снижение переобучения |
| `max_leaf_nodes` | Максимальное число листовых узлов | `None` | Жёсткое ограничение сложности дерева |
| `criterion` | Критерий разбиения | `"squared_error"` | Варианты: `squared_error`, `friedman_mse`, `absolute_error`, `poisson` |
| `splitter` | Стратегия выбора разбиения | `"best"` | `"best"` — лучшее разбиение; `"random"` — лучшее из случайных |
| `ccp_alpha` | Параметр Cost-Complexity Pruning | `0.0` | Больше значение — более сильная обрезка дерева после построения |

**Переобучение vs Недообучение:**
- **Глубокое дерево без ограничений** — запоминает обучающую выборку, плохо обобщает (переобучение).
- **Слишком мелкое дерево** — не улавливает закономерности (недообучение).
- **Оптимальное дерево** — баланс между сложностью модели и обобщающей способностью.

## Преимущества и недостатки

### Преимущества

| # | Преимущество | Пояснение |
|---|---|---|
| 1 | Интерпретируемость | Дерево легко визуализировать и объяснить заказчику |
| 2 | Не требует масштабирования | Нормализация или стандартизация признаков не нужна |
| 3 | Работа со смешанными типами | Может работать с числовыми и категориальными признаками |
| 4 | Нелинейные зависимости | Способно моделировать сложные нелинейные связи |
| 5 | Быстрое предсказание | $O(\log n)$ для сбалансированного дерева |
| 6 | Встроенный отбор признаков | Автоматически определяет важность признаков |

### Недостатки

| # | Недостаток | Пояснение |
|---|---|---|
| 1 | Склонность к переобучению | Без ограничений дерево запоминает обучающую выборку |
| 2 | Нестабильность | Небольшие изменения в данных могут дать совсем другое дерево |
| 3 | Ступенчатые предсказания | Предсказания — кусочно-постоянная функция, не гладкая |
| 4 | Жадный алгоритм | CART не гарантирует глобально оптимальное дерево |
| 5 | Проблема экстраполяции | Не может предсказывать значения за пределами обучающей выборки |

## GridSearchCV

**GridSearchCV (Grid Search with Cross-Validation)** — метод полного перебора заданной сетки параметров с оценкой каждой комбинации через кросс-валидацию.

**Алгоритм работы:**

1. Задаётся сетка параметров, например: `max_depth` = [3, 5, 10], `min_samples_leaf` = [1, 5, 10].
2. Перебираются **все** комбинации (в данном примере $3 \times 3 = 9$ комбинаций).
3. Для каждой комбинации данные делятся на $K$ фолдов (например, 5).
4. Модель обучается $K$ раз: каждый раз один фолд — валидационный, остальные — обучающие.
5. Результат — среднее качество по $K$ фолдам для каждой комбинации.
6. Выбирается комбинация с лучшим средним качеством.

**Почему ручной подбор глубины и GridSearchCV могут дать разные результаты:**

При ручном переборе изменяется только `max_depth`, а остальные параметры зафиксированы по умолчанию (например, `min_samples_leaf=1`). GridSearchCV перебирает комбинации нескольких параметров одновременно. Может оказаться, что `max_depth=20` в сочетании с `min_samples_leaf=10` работает лучше, чем `max_depth=8` с `min_samples_leaf=1`. Кроме того, GridSearchCV использует кросс-валидацию (усредняет по $K$ фолдам), тогда как ручной подбор — одно фиксированное разбиение train/test, что также может давать расхождение в результатах.

## Когда использовать

- Нелинейная зависимость между признаками и целевой переменной.
- Данные содержат взаимодействия признаков, которые трудно уловить линейными моделями.
- Требуется интерпретируемость предсказаний (возможность объяснить, почему модель приняла то или иное решение).
- Данные содержат признаки различных типов (числовые и категориальные) без необходимости предобработки.
- Задача не требует экстраполяции за пределы обучающей выборки.
- Модель используется как базовый элемент (base learner) для ансамблевых методов (Random Forest, Gradient Boosting).
