# Лабораторная 4 — Наивный Байес

## Теоретические основы

**Наивный байесовский классификатор** — семейство вероятностных классификаторов, основанных на **теореме Байеса** с дополнительным допущением об **условной независимости** признаков при заданном классе. Несмотря на то что в реальных данных признаки почти никогда не являются независимыми, данное допущение радикально упрощает вычисления и на практике приводит к конкурентоспособным результатам.

### Теорема Байеса и классификация

Теорема Байеса связывает **апостериорную** вероятность класса $y$ при наблюдаемых признаках $\mathbf{x} = (x_1, x_2, \ldots, x_n)$ с **априорной** вероятностью класса и **правдоподобием** данных:

$$P(y \mid x_1, x_2, \ldots, x_n) = \frac{P(y) \cdot P(x_1, x_2, \ldots, x_n \mid y)}{P(x_1, x_2, \ldots, x_n)}$$

Здесь:
- $P(y)$ — **априорная вероятность** класса (prior), оценённая по обучающей выборке;
- $P(x_1, \ldots, x_n \mid y)$ — **правдоподобие** (likelihood), вероятность наблюдать данный набор признаков при условии класса $y$;
- $P(x_1, \ldots, x_n)$ — **маргинальное правдоподобие** (evidence), нормировочная константа;
- $P(y \mid x_1, \ldots, x_n)$ — **апостериорная вероятность** (posterior).

### «Наивное» допущение

Вычисление совместного правдоподобия $P(x_1, \ldots, x_n \mid y)$ требует экспоненциально большого числа параметров. Наивное допущение полагает, что признаки **условно независимы** при заданном классе:

$$P(x_1, x_2, \ldots, x_n \mid y) = \prod_{i=1}^{n} P(x_i \mid y)$$

Это сводит задачу к оценке $n$ одномерных распределений вместо одного $n$-мерного.

### Почему метод работает, несмотря на допущение

Для задачи классификации не требуется точная оценка апостериорных вероятностей — достаточно правильного **ранжирования** классов. Даже при нарушении условия независимости произведение одномерных правдоподобий зачастую сохраняет верный порядок классов, что обеспечивает корректную классификацию.

### Интуитивный пример: классификация спама

Пусть требуется отнести письмо к классу «спам» или «не спам» по наличию слов «скидка», «бесплатно», «отчёт». Классификатор вычисляет:

$$P(\text{спам} \mid \text{скидка, бесплатно, отчёт}) \propto P(\text{спам}) \cdot P(\text{скидка} \mid \text{спам}) \cdot P(\text{бесплатно} \mid \text{спам}) \cdot P(\text{отчёт} \mid \text{спам})$$

и аналогичную величину для класса «не спам», после чего выбирает класс с большей апостериорной вероятностью.

## Математический аппарат

### Правило принятия решения

Поскольку знаменатель $P(x_1, \ldots, x_n)$ одинаков для всех классов, классификатор выбирает класс, максимизирующий числитель:

$$\hat{y} = \arg\max_{y} \; P(y) \prod_{i=1}^{n} P(x_i \mid y)$$

На практике используется **логарифмирование** для предотвращения вычислительного underflow:

$$\hat{y} = \arg\max_{y} \left[ \log P(y) + \sum_{i=1}^{n} \log P(x_i \mid y) \right]$$

### Разновидности наивного Байеса

#### GaussianNB (для непрерывных признаков)

Предполагается, что значения каждого признака внутри класса распределены нормально:

$$P(x_i \mid y) = \frac{1}{\sqrt{2\pi\sigma_{y,i}^{2}}} \exp\!\left(-\frac{(x_i - \mu_{y,i})^{2}}{2\sigma_{y,i}^{2}}\right)$$

где $\mu_{y,i}$ и $\sigma_{y,i}^{2}$ — среднее и дисперсия признака $x_i$ для объектов класса $y$, оценённые по обучающей выборке.

#### MultinomialNB (для счётных данных)

Применяется в задачах классификации текстов, где признаками являются частоты слов (bag-of-words, TF-IDF). Правдоподобие строится на основе мультиномиального распределения:

$$P(x_i \mid y) = \frac{N_{y,i} + \alpha}{N_y + \alpha \cdot n}$$

где $N_{y,i}$ — суммарное число вхождений признака $i$ среди объектов класса $y$, $N_y$ — общее число вхождений всех признаков в классе $y$, $n$ — число признаков, $\alpha$ — параметр сглаживания.

#### BernoulliNB (для бинарных признаков)

Каждый признак представлен как наличие (1) или отсутствие (0) некоторого свойства:

$$P(\mathbf{x} \mid y) = \prod_{i=1}^{n} \left[ P(x_i = 1 \mid y) \right]^{x_i} \cdot \left[ 1 - P(x_i = 1 \mid y) \right]^{(1 - x_i)}$$

В отличие от MultinomialNB, здесь **учитывается отсутствие** признака, что может быть информативным (например, отсутствие определённого слова в документе).

### Сглаживание Лапласа

Если некоторый признак $x_i$ не встречается ни в одном объекте класса $y$ в обучающей выборке, то $P(x_i \mid y) = 0$, что обнуляет всё произведение. Для предотвращения этого применяется **аддитивное сглаживание (Лапласа)**:

$$\hat{P}(x_i \mid y) = \frac{N_{y,i} + \alpha}{N_y + \alpha \cdot n}$$

При $\alpha = 1$ получается классическое сглаживание Лапласа; при $\alpha < 1$ — сглаживание Лидстоуна.

## Гиперпараметры

### MultinomialNB

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `alpha` | `float` | `1.0` | Параметр аддитивного сглаживания. Значение `0` — без сглаживания (риск нулевых вероятностей). Значения `0.01`–`1.0` наиболее распространены. |
| `fit_prior` | `bool` | `True` | Если `True`, априорные вероятности классов вычисляются по данным. Если `False`, используется равномерное распределение. |
| `class_prior` | `array` | `None` | Явное задание априорных вероятностей классов. Переопределяет `fit_prior`. |
| `force_alpha` | `bool` | `True` | Если `True`, альфа используется как есть; если `False`, при `alpha=0` может быть автоматически скорректирована. |

### GaussianNB

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `priors` | `array` | `None` | Априорные вероятности классов. Если не заданы, вычисляются по обучающей выборке. |
| `var_smoothing` | `float` | `1e-9` | Доля наибольшей дисперсии среди всех признаков, добавляемая к дисперсиям для стабильности вычислений. |

### BernoulliNB

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `alpha` | `float` | `1.0` | Параметр аддитивного сглаживания (аналогично MultinomialNB). |
| `binarize` | `float` | `0.0` | Порог бинаризации входных признаков. Если `None`, предполагается, что данные уже бинарные. |
| `fit_prior` | `bool` | `True` | Обучение априорных вероятностей по данным. |
| `class_prior` | `array` | `None` | Явное задание априорных вероятностей классов. |

## Преимущества и недостатки

### Преимущества

| Свойство | Пояснение |
|---|---|
| Высокая скорость обучения и предсказания | Сложность обучения $O(N \cdot n)$, предсказания $O(K \cdot n)$, где $N$ — размер выборки, $n$ — число признаков, $K$ — число классов |
| Хорошая работа на высокоразмерных данных | Не подвержен проклятию размерности благодаря оценке одномерных распределений |
| Малый объём обучающих данных | Требует меньше данных для оценки параметров, чем дискриминативные модели |
| Устойчивость к нерелевантным признакам | Нерелевантные признаки вносят примерно одинаковый вклад для всех классов и не влияют на решение |
| Вероятностные предсказания | Выдаёт оценки апостериорных вероятностей, а не только метки классов |
| Простота реализации и интерпретируемости | Параметры модели — оценки условных вероятностей, имеющие ясный смысл |

### Недостатки

| Свойство | Пояснение |
|---|---|
| Допущение о независимости признаков | Редко выполняется на практике; коррелированные признаки ухудшают калибровку вероятностей |
| Плохая калибровка вероятностей | Выходные значения $P(y \mid \mathbf{x})$ часто смещены к 0 и 1 |
| Чувствительность к балансу классов | При сильном дисбалансе априорные вероятности доминируют в решении |
| Ограниченная выразительность | Не способен моделировать сложные нелинейные границы решений |

---

## Когда использовать

| Сценарий | Рекомендация |
|---|---|
| Классификация текстов (спам-фильтрация, анализ тональности) | MultinomialNB — стандартный baseline |
| Данные с бинарными признаками | BernoulliNB |
| Непрерывные признаки с приблизительно нормальным распределением | GaussianNB |
| Необходим быстрый baseline | Любая разновидность |
| Малый объём обучающих данных | Наивный Байес предпочтительнее сложных моделей |
| Сложные зависимости между признаками | Следует рассмотреть другие методы (SVM, Random Forest, нейронные сети) |
| Требуется точная калибровка вероятностей | Следует применить калибровку (CalibratedClassifierCV) или использовать другую модель |

## Реализация в лабораторной работе

В ноутбуке `sentiment_nb.ipynb` решается задача **мультиклассовой классификации тональности** текстовых отзывов (негативный / нейтральный / позитивный) с использованием **MultinomialNB**:

1. Предобработка текстов (очистка, лемматизация с помощью `pymorphy3`).
2. Векторизация с использованием **TfidfVectorizer** (униграммы и биграммы).
3. Обучение модели `MultinomialNB(alpha=0.3)`.
4. Кросс-валидация и подбор гиперпараметра `alpha`.
5. Оценка качества по метрикам precision, recall, F1-score.
