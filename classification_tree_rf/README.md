# Классификационное дерево и Random Forest

## Теоретические основы

### Дерево решений (классификация)

**Дерево решений** (Decision Tree) - непараметрическая модель машинного обучения, осуществляющая классификацию путём последовательного разбиения пространства признаков на непересекающиеся области. Каждый внутренний узел дерева содержит условие вида $x_j \leq t$ (где $x_j$ - значение $j$-го признака, $t$ - порог), каждый лист содержит метку класса.

Построение дерева основано на принципе **рекурсивного бинарного разбиения** (recursive binary splitting):

1. Для текущего узла перебираются все признаки $j$ и все пороги $t$.
2. Выбирается пара $(j, t)$, минимизирующая выбранный критерий неоднородности (Gini или энтропию) в дочерних узлах.
3. Данные разбиваются на два подмножества: $\{x \mid x_j \leq t\}$ и $\{x \mid x_j > t\}$.
4. Процедура рекурсивно повторяется для каждого дочернего узла до выполнения критерия остановки (достигнута максимальная глубина, минимальное число объектов в узле и т.д.).
5. Листу присваивается метка наиболее частого класса среди попавших в него объектов.

**Пример: классификация заявок на кредит.** Пусть имеются признаки: доход, возраст, наличие залога. Дерево может построить следующую цепочку решений:
- Корень: «Доход > 50 000?»
  - Да: «Есть залог?»
    - Да: **Одобрить**
    - Нет: «Возраст > 25?»
      - Да: **Одобрить**
      - Нет: **Отказать**
  - Нет: **Отказать**

Каждое разбиение выбрано так, чтобы максимально «очистить» дочерние узлы, т.е. сделать их более однородными по классу.

### Random Forest

**Random Forest** (случайный лес) - ансамблевый метод, объединяющий множество деревьев решений для повышения точности и устойчивости к переобучению. Основан на принципе **бэггинга** (Bootstrap Aggregating, Breiman, 2001):

1. Из исходной обучающей выборки размера $n$ формируется $B$ **бутстрэп-выборок** - случайных выборок с возвращением того же размера $n$.
2. На каждой бутстрэп-выборке строится дерево решений. При этом в каждом узле для выбора разбиения рассматривается не весь набор признаков, а лишь **случайное подмножество** из $m$ признаков (обычно $m = \lfloor\sqrt{p}\rfloor$ для классификации, где $p$ - общее число признаков).
3. Итоговое предсказание формируется **голосованием большинством** (majority voting):

$$\hat{y}(x) = \arg\max_k \sum_{b=1}^{B} \mathbb{I}\bigl[\hat{y}_b(x) = k\bigr]$$

где $\hat{y}_b(x)$ - предсказание $b$-го дерева, $\mathbb{I}[\cdot]$ - индикаторная функция.

Случайность на двух уровнях (бутстрэп по объектам и случайный отбор признаков) обеспечивает **декорреляцию** деревьев в ансамбле. Это снижает дисперсию итогового предсказания без существенного увеличения смещения.

## Математический аппарат

### Критерии разбиения

Пусть узел $Q_m$ содержит $N_m$ объектов, из которых доля объектов класса $k$ равна:

$$p_{mk} = \frac{1}{N_m} \sum_{x_i \in Q_m} \mathbb{I}[y_i = k]$$

#### Примесь Джини (Gini Impurity)

$$Gini(Q_m) = 1 - \sum_{k=1}^{K} p_{mk}^2$$

Примесь Джини принимает значение 0, когда все объекты узла принадлежат одному классу (идеальная чистота), и максимальное значение $1 - 1/K$, когда классы представлены равномерно. Для бинарной классификации максимум равен 0.5.

#### Энтропия и Information Gain

$$H(Q_m) = -\sum_{k=1}^{K} p_{mk} \log_2 p_{mk}$$

Энтропия равна 0 при полной однородности узла и принимает максимальное значение $\log_2 K$ при равномерном распределении классов.

#### Формула выигрыша при разбиении

При разбиении узла $Q_m$ на дочерние узлы $Q_{left}$ и $Q_{right}$ выигрыш (Gain) вычисляется как:

$$Gain(Q_m, j, t) = I(Q_m) - \frac{N_{left}}{N_m} I(Q_{left}) - \frac{N_{right}}{N_m} I(Q_{right})$$

где $I(\cdot)$ - выбранная мера неоднородности (Gini или энтропия), $N_{left}$ и $N_{right}$ - число объектов в левом и правом дочерних узлах соответственно. Алгоритм выбирает разбиение $(j^*, t^*)$, максимизирующее $Gain$.

### Бэггинг (Bootstrap Aggregating)

#### Бутстрэп-выборка

Каждая бутстрэп-выборка $D_b$ формируется путём $n$ случайных извлечений с возвращением из исходной выборки $D = \{(x_1, y_1), \ldots, (x_n, y_n)\}$. Вероятность того, что конкретный объект **не попадёт** в бутстрэп-выборку:

$$P(\text{объект не выбран}) = \left(1 - \frac{1}{n}\right)^n \xrightarrow{n \to \infty} \frac{1}{e} \approx 0.368$$

Таким образом, в каждую бутстрэп-выборку попадает приблизительно 63.2% уникальных объектов исходной выборки.

#### Out-of-Bag (OOB) ошибка

Для каждого объекта $x_i$ определяется множество деревьев, при обучении которых этот объект **не** использовался:

$$\mathcal{B}_i = \{b : x_i \notin D_b\}$$

OOB-предсказание для объекта $x_i$:

$$\hat{y}_i^{OOB} = \arg\max_k \sum_{b \in \mathcal{B}_i} \mathbb{I}[\hat{y}_b(x_i) = k]$$

OOB-ошибка:

$$\varepsilon_{OOB} = \frac{1}{n} \sum_{i=1}^{n} \mathbb{I}[\hat{y}_i^{OOB} \neq y_i]$$

Данная оценка является несмещённой и по свойствам аналогична кросс-валидации, что позволяет оценить качество модели без выделения отдельной валидационной выборки.

## Гиперпараметры

### Дерево решений (DecisionTreeClassifier)

| Гиперпараметр | Описание | Значение по умолчанию | Типичный диапазон |
|---|---|---|---|
| `criterion` | Критерий разбиения: `"gini"` или `"entropy"` | `"gini"` | - |
| `max_depth` | Максимальная глубина дерева. Ограничивает сложность модели | `None` (без ограничений) | 2–20 |
| `min_samples_split` | Минимальное число объектов в узле для выполнения разбиения | 2 | 2–50 |
| `min_samples_leaf` | Минимальное число объектов в листовом узле | 1 | 1–20 |
| `max_features` | Число признаков, рассматриваемых при каждом разбиении | `None` (все признаки) | `"sqrt"`, `"log2"`, int, float |
| `ccp_alpha` | Параметр сложности для минимально-стоимостной обрезки (Cost-Complexity Pruning). Чем больше значение, тем сильнее обрезка | 0.0 | 0.0–0.05 |

### Random Forest (RandomForestClassifier)

| Гиперпараметр | Описание | Значение по умолчанию | Типичный диапазон |
|---|---|---|---|
| `n_estimators` | Число деревьев в ансамбле | 100 | 50–1000 |
| `max_features` | Число признаков для рассмотрения в каждом разбиении | `"sqrt"` | `"sqrt"`, `"log2"`, int, float |
| `bootstrap` | Использовать ли бутстрэп-выборки | `True` | `True` / `False` |
| `oob_score` | Вычислять ли OOB-ошибку (требует `bootstrap=True`) | `False` | `True` / `False` |
| `max_depth` | Максимальная глубина каждого дерева | `None` | 5–30 |
| `min_samples_split` | Минимальное число объектов для разбиения узла | 2 | 2–20 |
| `min_samples_leaf` | Минимальное число объектов в листе | 1 | 1–10 |
| `criterion` | Критерий разбиения | `"gini"` | `"gini"`, `"entropy"` |
| `ccp_alpha` | Параметр обрезки для каждого дерева | 0.0 | 0.0–0.05 |

## Преимущества и недостатки

### Дерево решений

| Преимущества | Недостатки |
|---|---|
| Высокая интерпретируемость: дерево легко визуализировать и объяснить | Склонность к переобучению, особенно при большой глубине |
| Не требует масштабирования признаков | Нестабильность: малые изменения данных приводят к существенно иному дереву |
| Работает с категориальными и числовыми признаками | Жадный алгоритм: оптимальное разбиение на каждом шаге не гарантирует глобальный оптимум |
| Нелинейные границы решений | Ступенчатые границы решений (не гладкие) |
| Быстрое обучение и предсказание | Плохо работает при несбалансированных классах без дополнительных мер |
| Автоматический отбор признаков | Высокая дисперсия (high variance) |

### Random Forest

| Преимущества | Недостатки |
|---|---|
| Значительно меньшая склонность к переобучению по сравнению с одиночным деревом | Потеря интерпретируемости: ансамбль из сотен деревьев невозможно визуализировать целиком |
| Высокая точность «из коробки» с минимальной настройкой | Значительно большее время обучения и предсказания |
| Встроенная оценка качества (OOB-ошибка) | Высокое потребление памяти при большом числе деревьев |
| Устойчивость к шуму и выбросам | Не экстраполирует: не может предсказывать значения за пределами обучающей выборки |
| Оценка важности признаков (feature importance) | Гиперпараметр `n_estimators` увеличивает вычислительные затраты |
| Эффективно параллелизуется | На сильно разреженных данных малого объёма может уступать линейным моделям |

## Когда использовать

**Дерево решений** целесообразно применять, когда:
- Требуется максимальная интерпретируемость модели (медицина, финансы, юриспруденция).
- Задача имеет небольшую размерность и объём данных.
- Модель используется как визуальный инструмент для анализа правил принятия решений.
- Необходим быстрый baseline для задачи классификации.

**Random Forest** целесообразно применять, когда:
- Данные содержат нелинейные зависимости и взаимодействия признаков.
- Высокая размерность пространства признаков.
- Необходима устойчивость к переобучению без сложной настройки.
- Требуется оценка важности признаков для последующего отбора.
- Имеется достаточно вычислительных ресурсов для обучения ансамбля.
- Необходим надёжный метод с минимальной предобработкой данных (не требует масштабирования, устойчив к выбросам).
