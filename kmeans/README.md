# Лабораторная 8 - K-Means, K-Means++

## Теоретические основы

**K-Means** - один из наиболее распространённых алгоритмов кластеризации, относящийся к классу методов разбиения (partitioning methods). Алгоритм разделяет множество из $n$ наблюдений на $K$ непересекающихся кластеров, где каждое наблюдение принадлежит кластеру с ближайшим центроидом.

Основная идея алгоритма заключается в итеративном чередовании двух шагов:
1. **Назначение** - каждая точка приписывается к кластеру с ближайшим центроидом.
2. **Обновление** - центроиды пересчитываются как средние арифметические точек, принадлежащих кластеру.

Процедура повторяется до тех пор, пока назначения не перестанут изменяться (или пока изменение целевой функции не станет меньше заданного порога).

### Интуитивный пример

Рассмотрим задачу кластеризации рукописных цифр (датасет `sklearn.datasets.load_digits`). Каждое изображение 8x8 пикселей представляется вектором из 64 признаков. K-Means работает в этом 64-мерном пространстве: находит 10 центроидов, каждый из которых является «усреднённым образом» цифры. Точки, близкие к одному центроиду, формируют кластер - группу похожих цифр. При этом алгоритм не знает истинных меток; он обнаруживает структуру данных исключительно на основе геометрической близости.

## Математический аппарат

### Целевая функция (инерция)

K-Means минимизирует **внутрикластерную сумму квадратов** (Within-Cluster Sum of Squares, WCSS), также называемую инерцией:

$$J = \sum_{j=1}^{K} \sum_{x_i \in C_j} \|x_i - \mu_j\|^2$$

где:
- $K$ - число кластеров,
- $C_j$ - множество точек, отнесённых к $j$-му кластеру,
- $\mu_j$ - центроид $j$-го кластера,
- $\|x_i - \mu_j\|^2$ - квадрат евклидова расстояния от точки $x_i$ до центроида $\mu_j$.

Задача является NP-трудной в общем случае. Алгоритм Ллойда находит локальный минимум.

### Алгоритм K-Means (Lloyd's algorithm)

**Вход:** множество точек $X = \{x_1, x_2, \ldots, x_n\}$, число кластеров $K$.

**Шаг 0. Инициализация.** Выбрать $K$ начальных центроидов $\mu_1^{(0)}, \mu_2^{(0)}, \ldots, \mu_K^{(0)}$ (например, случайным образом из $X$).

**Шаг 1. Назначение (Assignment step).** Для каждой точки $x_i$ определить ближайший центроид:

$$C_j^{(t)} = \{x_i : \|x_i - \mu_j^{(t)}\| \leq \|x_i - \mu_l^{(t)}\|, \; \forall l = 1, \ldots, K\}$$

**Шаг 2. Обновление (Update step).** Пересчитать центроиды как средние точек кластера:

$$\mu_j^{(t+1)} = \frac{1}{|C_j^{(t)}|} \sum_{x_i \in C_j^{(t)}} x_i$$

**Шаг 3. Проверка сходимости.** Если назначения не изменились ($C_j^{(t+1)} = C_j^{(t)}$ для всех $j$) или $|J^{(t)} - J^{(t+1)}| < \varepsilon$, алгоритм завершается. Иначе - возврат к Шагу 1.

**Свойства:**
- Значение целевой функции $J$ монотонно убывает на каждой итерации.
- Алгоритм гарантированно сходится за конечное число шагов (число возможных разбиений конечно).
- Результат зависит от начальной инициализации - возможна сходимость к локальному минимуму.

### K-Means++ инициализация

Стандартная случайная инициализация может привести к неудачному начальному расположению центроидов и, как следствие, к плохому локальному минимуму. Метод **K-Means++** (Arthur, Vassilvitskii, 2007) решает эту проблему, выбирая начальные центроиды так, чтобы они были максимально удалены друг от друга.

**Алгоритм:**

1. Выбрать первый центроид $\mu_1$ равномерно случайно из $X$.
2. Для каждой точки $x_i \in X$ вычислить расстояние до ближайшего уже выбранного центроида:

$$D(x_i) = \min_{j \in \{1, \ldots, c\}} \|x_i - \mu_j\|$$

где $c$ - число уже выбранных центроидов.

3. Выбрать следующий центроид $\mu_{c+1}$ с вероятностью, пропорциональной $D(x_i)^2$:

$$P(x_i) = \frac{D(x_i)^2}{\sum_{x \in X} D(x)^2}$$

4. Повторять шаги 2–3, пока не будет выбрано $K$ центроидов.

**Теоретическая гарантия:** K-Means++ обеспечивает начальное приближение, для которого ожидаемое значение целевой функции не превышает $O(\log K)$ от оптимального. На практике это приводит к более быстрой сходимости и лучшему качеству кластеризации.

### Выбор оптимального K

#### Метод локтя (Elbow method)

Строится график зависимости инерции $J$ от числа кластеров $K$. По мере увеличения $K$ инерция монотонно убывает. Оптимальное $K$ соответствует точке «перегиба» (локтя), после которой уменьшение $J$ замедляется.

На практике перегиб не всегда выражен отчётливо (например, на датасете Digits кривая убывает плавно), что затрудняет выбор $K$ только по этому методу.

#### Коэффициент силуэта (Silhouette score)

Для каждой точки $x_i$ определяется **силуэтный коэффициент**:

$$s(i) = \frac{b(i) - a(i)}{\max(a(i), b(i))}$$

где:
- $a(i)$ - среднее расстояние от $x_i$ до остальных точек **того же** кластера (внутрикластерное расстояние),
- $b(i)$ - среднее расстояние от $x_i$ до точек **ближайшего чужого** кластера (межкластерное расстояние).

Значение $s(i) \in [-1, 1]$:

| Значение $s(i)$ | Интерпретация |
|:---:|:---|
| $\approx 1$ | Точка хорошо принадлежит своему кластеру |
| $\approx 0$ | Точка находится на границе между кластерами |
| $< 0$ | Точка, вероятно, отнесена к неверному кластеру |

Средний силуэтный коэффициент по всем точкам используется как общая мера качества кластеризации. Оптимальное $K$ - то, при котором средний силуэт максимален.

## Гиперпараметры

Основные гиперпараметры `sklearn.cluster.KMeans`:

| Параметр | Тип | По умолчанию | Описание |
|:---|:---:|:---:|:---|
| `n_clusters` | `int` | `8` | Число кластеров $K$ |
| `init` | `str` | `'k-means++'` | Метод инициализации центроидов: `'k-means++'` или `'random'` |
| `n_init` | `int` | `10` | Число запусков алгоритма с разными начальными центроидами; выбирается лучший результат по инерции |
| `max_iter` | `int` | `300` | Максимальное число итераций за один запуск |
| `tol` | `float` | `1e-4` | Порог сходимости: алгоритм останавливается, если относительное изменение инерции меньше `tol` |
| `algorithm` | `str` | `'lloyd'` | Реализация: `'lloyd'` (классический), `'elkan'` (ускоренный для евклидова расстояния) |
| `random_state` | `int` | `None` | Фиксация генератора случайных чисел для воспроизводимости |

**Рекомендации по подбору:**
- `n_clusters` - подбирается методом локтя и/или силуэтным анализом.
- `init='k-means++'` - предпочтительнее `'random'` в подавляющем большинстве случаев.
- `n_init=10` - достаточно для большинства задач; увеличение повышает устойчивость, но замедляет обучение.

## Преимущества и недостатки

### Преимущества

| Свойство | Пояснение |
|:---|:---|
| Простота реализации | Алгоритм интуитивно понятен и легко реализуется |
| Масштабируемость | Временная сложность $O(n \cdot K \cdot d \cdot T)$, где $d$ - размерность, $T$ - число итераций; линейна по $n$ |
| Скорость | Один из самых быстрых алгоритмов кластеризации |
| Гарантированная сходимость | Алгоритм всегда сходится за конечное число итераций |
| Интерпретируемость | Центроиды имеют наглядный смысл (например, «средняя цифра» в задаче кластеризации рукописных символов) |

### Недостатки

| Свойство | Пояснение |
|:---|:---|
| Необходимость задавать $K$ | Число кластеров должно быть известно заранее |
| Чувствительность к инициализации | Случайная инициализация может приводить к плохим локальным минимумам (частично решается K-Means++) |
| Предположение о форме кластеров | Алгоритм предполагает выпуклые (приблизительно сферические) кластеры равного размера |
| Чувствительность к выбросам | Выбросы сильно влияют на положение центроидов |
| Чувствительность к масштабу признаков | Евклидово расстояние зависит от масштаба; необходима стандартизация |
| Локальный оптимум | Нет гарантии нахождения глобального минимума $J$ |

## Когда использовать

K-Means целесообразно применять при выполнении следующих условий:

- **Число кластеров известно** или может быть обоснованно подобрано (метод локтя, силуэтный анализ).
- **Кластеры имеют приблизительно сферическую форму** и сопоставимый размер.
- **Данные не содержат значительного числа выбросов** (либо выбросы предварительно удалены).
- **Требуется высокая скорость** - алгоритм хорошо масштабируется на большие датасеты.
- **Признаки предварительно стандартизированы** - все признаки приведены к единому масштабу.

K-Means **не рекомендуется**, если кластеры имеют сложную форму (цепочки, полумесяцы), существенно различаются по плотности или размеру, либо если данные содержат большое количество шума. В таких случаях следует рассмотреть DBSCAN, Mean Shift или спектральную кластеризацию.
