# Лабораторная 10 - Метод главных компонент (PCA)

## Описание алгоритма

**PCA (Principal Component Analysis)** - метод снижения размерности, основанный на ортогональной проекции данных на новые оси, называемые **главными компонентами**. Главные компоненты упорядочены по убыванию объяснённой дисперсии: первая компонента захватывает максимальную долю вариативности данных, вторая - максимум оставшейся дисперсии при условии ортогональности первой, и так далее.

PCA не является алгоритмом обучения с учителем - он работает исключительно с матрицей признаков $\mathbf{X}$ без использования целевой переменной. Формально задача сводится к нахождению такого ортогонального преобразования координат, при котором наибольшая дисперсия данных приходится на первые оси нового пространства.

### Интуитивный пример

Рассмотрим набор изображений размером $64 \times 64$ пикселей. Каждое изображение описывается вектором из $4096$ признаков (значения яркости пикселей). Однако большинство пикселей сильно коррелированы друг с другом: соседние области лица, фона и т.д. меняются согласованно. PCA позволяет найти $d \ll 4096$ линейных комбинаций исходных признаков, которые сохраняют основную структуру данных и позволяют восстановить изображение с приемлемым качеством. Например, для датасета Olivetti Faces достаточно $\approx 100$ компонент, чтобы сохранить $\geq 95\%$ дисперсии при исходных $4096$ признаках.

## Теоретические основы

### Постановка задачи

Дана матрица данных $\mathbf{X} \in \mathbb{R}^{n \times p}$, где $n$ - число наблюдений, $p$ - число признаков. Требуется найти линейное преобразование $\mathbf{Z} = \mathbf{X} \mathbf{V}_d$, где $\mathbf{V}_d \in \mathbb{R}^{p \times d}$, $d < p$, такое что проекция $\mathbf{Z} \in \mathbb{R}^{n \times d}$ сохраняет максимум дисперсии исходных данных.

### Связь с собственным разложением ковариационной матрицы

Решение задачи PCA эквивалентно нахождению собственных векторов ковариационной матрицы $\Sigma$, соответствующих наибольшим собственным значениям. Столбцы $\mathbf{V}_d$ - это первые $d$ собственных векторов матрицы $\Sigma$, а соответствующие собственные значения $\lambda_1, \lambda_2, \ldots, \lambda_d$ определяют долю дисперсии, захваченную каждой компонентой.

---

## Математический аппарат

### 1. Центрирование данных

Перед вычислением PCA данные необходимо центрировать:

$$\bar{\mathbf{x}} = \frac{1}{n} \sum_{i=1}^{n} \mathbf{x}_i, \qquad \tilde{\mathbf{X}} = \mathbf{X} - \mathbf{1}_n \bar{\mathbf{x}}^T$$

где $\mathbf{1}_n$ - вектор-столбец из единиц длины $n$. Далее под $\mathbf{X}$ подразумевается центрированная матрица.

### 2. Ковариационная матрица

Выборочная ковариационная матрица центрированных данных:

$$\Sigma = \frac{1}{n-1} \mathbf{X}^T \mathbf{X} \in \mathbb{R}^{p \times p}$$

Элемент $\Sigma_{ij}$ описывает линейную взаимосвязь между $i$-м и $j$-м признаками. Матрица $\Sigma$ симметрична и положительно полуопределена.

### 3. Собственные значения и собственные векторы

Решается задача на собственные значения:

$$\Sigma \mathbf{v}_k = \lambda_k \mathbf{v}_k, \qquad k = 1, 2, \ldots, p$$

где:

- $\lambda_1 \geq \lambda_2 \geq \ldots \geq \lambda_p \geq 0$ - собственные значения, упорядоченные по убыванию;
- $\mathbf{v}_1, \mathbf{v}_2, \ldots, \mathbf{v}_p$ - соответствующие ортонормированные собственные векторы ($\mathbf{v}_i^T \mathbf{v}_j = \delta_{ij}$).

Каждый собственный вектор $\mathbf{v}_k$ задаёт направление $k$-й главной компоненты, а собственное значение $\lambda_k$ равно дисперсии данных вдоль этого направления.

### 4. Проекция данных

Проекция на $d$ главных компонент:

$$\mathbf{Z} = \mathbf{X} \mathbf{V}_d, \qquad \mathbf{V}_d = [\mathbf{v}_1 \mid \mathbf{v}_2 \mid \ldots \mid \mathbf{v}_d] \in \mathbb{R}^{p \times d}$$

Результат - матрица $\mathbf{Z} \in \mathbb{R}^{n \times d}$, где каждый столбец $\mathbf{z}_k = \mathbf{X} \mathbf{v}_k$ представляет значения $k$-й главной компоненты для всех наблюдений.

### 5. Восстановление данных

Приближённое восстановление исходных данных из сжатого представления:

$$\hat{\mathbf{X}} = \mathbf{Z} \mathbf{V}_d^T = \mathbf{X} \mathbf{V}_d \mathbf{V}_d^T$$

Ошибка восстановления (в смысле нормы Фробениуса) равна:

$$\| \mathbf{X} - \hat{\mathbf{X}} \|_F^2 = \sum_{k=d+1}^{p} \lambda_k$$

### 6. Объяснённая дисперсия

Доля дисперсии, объяснённая $k$-й компонентой:

$$\text{explained\_variance\_ratio}_k = \frac{\lambda_k}{\displaystyle\sum_{i=1}^{p} \lambda_i}$$

Кумулятивная объяснённая дисперсия первых $d$ компонент:

$$\text{cumulative\_variance}(d) = \sum_{k=1}^{d} \frac{\lambda_k}{\displaystyle\sum_{i=1}^{p} \lambda_i}$$

### 7. Выбор числа компонент

Основные критерии выбора $d$:

| Критерий | Описание | Формула / правило |
|---|---|---|
| Порог кумулятивной дисперсии | Выбрать минимальное $d$, при котором сохраняется заданная доля дисперсии | $\text{cumulative\_variance}(d) \geq \theta$, обычно $\theta = 0.95$ |
| Scree plot (метод «каменистой осыпи») | На графике $\lambda_k$ от $k$ найти «локоть» - точку резкого замедления убывания | Визуальный анализ |
| Критерий Кайзера | Оставить компоненты с $\lambda_k > 1$ (для стандартизированных данных) | $\lambda_k > 1$ |
| Автоматический (MLE) | Метод максимального правдоподобия Minka | `n_components='mle'` в sklearn |

## Важность стандартизации

PCA максимизирует дисперсию. Если признаки измерены в разных шкалах (например, рост в сантиметрах и вес в килограммах), признак с большим разбросом значений будет доминировать в первых компонентах, независимо от его реальной информативности.

**Стандартизация обязательна перед PCA.** Применяется `StandardScaler`:

$$x_j^{\text{scaled}} = \frac{x_j - \mu_j}{\sigma_j}$$

где $\mu_j$ и $\sigma_j$ - среднее и стандартное отклонение $j$-го признака. После стандартизации все признаки имеют нулевое среднее и единичную дисперсию, и ковариационная матрица совпадает с корреляционной матрицей.

| Ситуация | Без стандартизации | Со стандартизацией |
|---|---|---|
| Признаки в одной шкале | Допустимо | Рекомендуется |
| Признаки в разных шкалах | Некорректно - доминирует признак с наибольшим разбросом | Корректно - все признаки вносят равный вклад |
| Интерпретация компонент | Зависит от единиц измерения | Инвариантна к масштабу |

## Гиперпараметры (`sklearn.decomposition.PCA`)

| Параметр | Тип | Значение по умолчанию | Описание |
|---|---|---|---|
| `n_components` | `int`, `float`, `'mle'`, `None` | `None` | Число компонент. `int` - точное число; `float` в $(0, 1)$ - минимальное число компонент для заданной доли дисперсии; `'mle'` - автоматический выбор по критерию Minka; `None` - все компоненты |
| `svd_solver` | `str` | `'auto'` | Алгоритм SVD-разложения: `'full'` - полное разложение (LAPACK); `'arpack'` - усечённое (ARPACK, для больших разреженных матриц); `'randomized'` - рандомизированное (эффективно при $d \ll p$); `'auto'` - автоматический выбор |
| `whiten` | `bool` | `False` | Если `True`, компоненты масштабируются до единичной дисперсии: $\mathbf{z}_k \leftarrow \mathbf{z}_k / \sqrt{\lambda_k}$. Полезно перед алгоритмами, чувствительными к масштабу (например, SVM, KNN) |
| `tol` | `float` | `0.0` | Порог сходимости для `svd_solver='arpack'` |
| `random_state` | `int` или `None` | `None` | Зерно генератора для `svd_solver='randomized'` и `'arpack'` |

## Преимущества и недостатки

### Преимущества

| Свойство | Описание |
|---|---|
| Снижение размерности | Позволяет визуализировать многомерные данные в 2D/3D |
| Устранение мультиколлинеарности | Компоненты ортогональны - линейная зависимость между признаками устраняется |
| Ускорение обучения | Уменьшение числа признаков сокращает время обучения последующих моделей |
| Фильтрация шума | Отбрасывание компонент с малой дисперсией удаляет часть шума |
| Вычислительная эффективность | Сводится к SVD-разложению - хорошо оптимизированные алгоритмы, $O(np \min(n,p))$ |
| Отсутствие гиперпараметров модели | Единственный выбор - число компонент $d$; сам метод детерминирован |

### Недостатки

| Свойство | Описание |
|---|---|
| Только линейные зависимости | Не способен обнаружить нелинейную структуру данных (для этого существуют Kernel PCA, t-SNE, UMAP) |
| Потеря интерпретируемости | Компоненты - линейные комбинации всех исходных признаков; их смысл неочевиден |
| Чувствительность к выбросам | Выбросы сильно влияют на ковариационную матрицу и, следовательно, на направления компонент |
| Потеря информации | При $d < p$ часть информации неизбежно теряется; для некоторых задач это критично |
| Требует стандартизации | Без предварительного масштабирования результат некорректен |

---

## Когда использовать PCA

| Сценарий | Пример |
|---|---|
| Визуализация многомерных данных | Проекция датасета Iris (4 признака) на 2D-плоскость для визуальной оценки разделимости классов |
| Предобработка перед обучением модели | Снижение размерности с $p = 500$ до $d = 50$ перед подачей в SVM или KNN для ускорения и улучшения обобщения |
| Устранение мультиколлинеарности | Замена коррелированных финансовых показателей ортогональными компонентами перед линейной регрессией |
| Сжатие данных | Хранение изображений в сжатом виде ($d$ коэффициентов вместо $p$ пикселей) с возможностью восстановления |
| Анализ структуры данных | Исследование того, какие признаки наиболее сильно влияют на первые компоненты (`pca.components_`) |
