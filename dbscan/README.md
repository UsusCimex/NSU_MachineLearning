# Лабораторная 9 - DBSCAN

## Описание алгоритма

**DBSCAN (Density-Based Spatial Clustering of Applications with Noise)** - алгоритм кластеризации, основанный на анализе плотности распределения точек в пространстве признаков. Алгоритм был предложен Эстером и др. в 1996 году и остаётся одним из наиболее цитируемых методов кластеризации в литературе по машинному обучению.

В отличие от методов, основанных на разбиении пространства (K-Means), DBSCAN не требует априорного задания числа кластеров и способен обнаруживать кластеры произвольной формы. Основная идея состоит в том, что кластер представляет собой область высокой плотности точек, отделённую от других кластеров областями низкой плотности.

### Интуитивный пример

Рассмотрим множество точек на плоскости. Если выбрать произвольную точку и провести вокруг неё окружность радиуса $\varepsilon$, то в плотной области внутри окружности окажется много точек. DBSCAN использует этот принцип: если в $\varepsilon$-окрестности точки находится достаточное количество соседей (не менее `min_samples`), эта точка считается ядровой и формирует основу кластера. Кластер затем «растёт», последовательно присоединяя все точки, достижимые по плотности из ядровых точек. Точки, не достижимые ни из одного кластера, объявляются шумом.

## Теоретические основы

### Типы точек

DBSCAN классифицирует каждую точку множества данных в одну из трёх категорий:

| Тип точки | Определение | Роль в кластеризации |
|---|---|---|
| **Ядровая (core point)** | Точка $x$, в $\varepsilon$-окрестности которой находится не менее `min_samples` точек | Формирует «скелет» кластера; каждая ядровая точка гарантированно принадлежит некоторому кластеру |
| **Граничная (border point)** | Точка, попадающая в $\varepsilon$-окрестность хотя бы одной ядровой точки, но сама не являющаяся ядровой | Принадлежит кластеру, но не может расширять его далее |
| **Шумовая (noise point)** | Точка, не попадающая в $\varepsilon$-окрестность ни одной ядровой точки | Не принадлежит ни одному кластеру; трактуется как выброс |

### Отношения плотности

Формальное описание алгоритма опирается на три ключевых отношения между точками:

**Прямая достижимость по плотности (directly density-reachable).** Точка $x'$ прямо достижима по плотности из точки $x$ относительно параметров $\varepsilon$ и `min_samples`, если выполнены два условия:

$$x' \in N_\varepsilon(x)$$

$$|N_\varepsilon(x)| \geq \text{min\_samples}$$

то есть $x$ является ядровой точкой и $x'$ находится в её $\varepsilon$-окрестности. Данное отношение **несимметрично**: если $x'$ - граничная точка, она прямо достижима из ядровой $x$, но $x$ не обязательно прямо достижима из $x'$.

**Достижимость по плотности (density-reachable).** Точка $x_n$ достижима по плотности из точки $x_1$ относительно $\varepsilon$ и `min_samples`, если существует цепочка точек $x_1, x_2, \ldots, x_n$, в которой каждая последующая точка прямо достижима по плотности из предыдущей:

$$\forall i \in \{1, \ldots, n-1\}: x_{i+1} \text{ прямо достижима по плотности из } x_i$$

Данное отношение является **транзитивным**, но по-прежнему **несимметричным**.

**Связанность по плотности (density-connected).** Две точки $x$ и $y$ связаны по плотности относительно $\varepsilon$ и `min_samples`, если существует такая точка $o$, из которой обе точки достижимы по плотности:

$$\exists \, o \in D: \quad x \text{ достижима из } o \quad \wedge \quad y \text{ достижима из } o$$

Это отношение **симметрично**. Кластер в DBSCAN определяется как максимальное множество точек, попарно связанных по плотности.

### Ключевые свойства

- **Не требует задания числа кластеров.** Количество кластеров определяется автоматически структурой данных
- **Автоматическое обнаружение выбросов.** Шумовые точки естественным образом отделяются от кластеров
- **Кластеры произвольной формы.** Алгоритм не предполагает сферичность или выпуклость кластеров
- **Детерминированность для ядровых точек.** Принадлежность ядровых точек к кластерам не зависит от порядка обхода данных (граничные точки могут быть отнесены к разным кластерам при разном порядке обхода)

## Математический аппарат

### Определения

**$\varepsilon$-окрестность** точки $x$ - множество всех точек датасета $D$, расстояние до которых не превышает $\varepsilon$:

$$N_\varepsilon(x) = \{x' \in D : d(x, x') \leq \varepsilon\}$$

где $d(x, x')$ - функция расстояния (метрика). По умолчанию используется евклидово расстояние:

$$d(x, x') = \sqrt{\sum_{i=1}^{n} (x_i - x'_i)^2}$$

**Ядровая точка (core point)** - точка $x$, для которой мощность $\varepsilon$-окрестности не меньше заданного порога:

$$|N_\varepsilon(x)| \geq \text{min\_samples}$$

### Алгоритм DBSCAN

Алгоритм выполняется следующим образом:

1. **Инициализация.** Пометить все точки как непосещённые. Установить текущий идентификатор кластера $C = 0$.
2. **Итерация по точкам.** Для каждой непосещённой точки $x \in D$:
   - Пометить $x$ как посещённую
   - Вычислить $\varepsilon$-окрестность: $N_\varepsilon(x)$
   - Если $|N_\varepsilon(x)| < \text{min\_samples}$, пометить $x$ как шумовую (эта метка может быть изменена позже, если $x$ окажется граничной точкой другого кластера)
   - Если $|N_\varepsilon(x)| \geq \text{min\_samples}$:
     - Увеличить идентификатор кластера: $C = C + 1$
     - Вызвать процедуру расширения кластера `ExpandCluster(x, N_\varepsilon(x), C)`
3. **Процедура `ExpandCluster(x, N, C)`:**
   - Присвоить точке $x$ кластер $C$
   - Для каждой точки $x' \in N$:
     - Если $x'$ не посещена:
       - Пометить $x'$ как посещённую
       - Вычислить $N_\varepsilon(x')$
       - Если $|N_\varepsilon(x')| \geq \text{min\_samples}$, добавить точки из $N_\varepsilon(x')$ к множеству $N$
     - Если $x'$ ещё не принадлежит ни одному кластеру, присвоить ей кластер $C$

### Вычислительная сложность

| Операция | Без индекса | С пространственным индексом (R-tree, KD-tree, Ball-tree) |
|---|---|---|
| Запрос $\varepsilon$-окрестности | $O(n)$ | $O(\log n)$ |
| Общая сложность алгоритма | $O(n^2)$ | $O(n \log n)$ |
| Требования к памяти | $O(n)$ | $O(n)$ |

где $n$ - количество точек в датасете.

### Подбор параметров

#### Метод k-расстояний для выбора $\varepsilon$

Основной метод подбора параметра $\varepsilon$ - построение графика k-расстояний (k-distance graph):

1. Выбрать значение $k = \text{min\_samples} - 1$ (или $k = \text{min\_samples}$)
2. Для каждой точки $x_i \in D$ вычислить расстояние до $k$-го ближайшего соседа: $d_k(x_i)$
3. Отсортировать расстояния $d_k(x_1), d_k(x_2), \ldots, d_k(x_n)$ по убыванию
4. Построить график отсортированных расстояний
5. Найти «колено» (knee/elbow) графика - точку, в которой кривизна максимальна. Значение $\varepsilon$ соответствует ординате этой точки

Интуиция: для точек внутри кластеров k-расстояния будут малы и примерно одинаковы. Для шумовых точек расстояния будут значительно больше. Точка перегиба на графике отделяет одну группу от другой.

#### Подбор `min_samples`

Рекомендации по выбору параметра `min_samples`:

| Правило | Описание |
|---|---|
| $\text{min\_samples} \geq D + 1$ | Где $D$ - размерность пространства признаков. Минимально допустимое значение |
| $\text{min\_samples} = 2 \cdot D$ | Практическая рекомендация для большинства случаев |
| Увеличение `min_samples` | Даёт более консервативную кластеризацию: больше точек помечается как шум |
| `min_samples = 1` | Вырожденный случай: каждая точка - ядровая, шумовых точек нет |

## Гиперпараметры

Полный перечень параметров `sklearn.cluster.DBSCAN`:

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `eps` | `float` | `0.5` | Радиус $\varepsilon$-окрестности. Определяет максимальное расстояние между двумя точками, при котором одна считается соседом другой |
| `min_samples` | `int` | `5` | Минимальное число точек в $\varepsilon$-окрестности для признания точки ядровой |
| `metric` | `str` | `'euclidean'` | Метрика расстояния. Допустимые значения: `'euclidean'`, `'manhattan'`, `'cosine'`, `'chebyshev'`, `'minkowski'` и другие метрики из `sklearn.metrics.pairwise_distances` |
| `algorithm` | `str` | `'auto'` | Алгоритм поиска ближайших соседей: `'auto'`, `'ball_tree'`, `'kd_tree'`, `'brute'` |
| `leaf_size` | `int` | `30` | Размер листа для `ball_tree` и `kd_tree`. Влияет на скорость построения дерева и запросов, а также на потребление памяти |
| `p` | `float` | `None` | Параметр степени для метрики Минковского: $d(x, y) = \left(\sum_i |x_i - y_i|^p\right)^{1/p}$. При $p=2$ - евклидово расстояние, при $p=1$ - манхэттенское |
| `n_jobs` | `int` | `None` | Количество параллельных процессов. `-1` - использовать все ядра |

## Преимущества и недостатки

### Преимущества

| Свойство | Пояснение |
|---|---|
| Не требует задания числа кластеров | Количество кластеров определяется автоматически на основе структуры данных |
| Обнаружение выбросов | Шумовые точки выявляются как побочный результат работы алгоритма |
| Произвольная форма кластеров | Способен находить кластеры любой геометрической формы (в отличие от K-Means, ограниченного выпуклыми кластерами) |
| Устойчивость к выбросам | Выбросы не влияют на формирование кластеров |
| Единственный проход по данным | При использовании пространственного индекса работает за $O(n \log n)$ |

### Недостатки

| Свойство | Пояснение |
|---|---|
| Чувствительность к параметрам | Качество кластеризации существенно зависит от выбора $\varepsilon$ и `min_samples` |
| Кластеры разной плотности | Не способен корректно разделить кластеры с существенно различающейся плотностью (для этого существует OPTICS) |
| Высокая размерность | В пространствах высокой размерности понятие «плотности» теряет смысл (проклятие размерности); евклидово расстояние становится неинформативным |
| Граничные точки | Принадлежность граничных точек к кластерам может зависеть от порядка обхода данных (недетерминированность) |
| Масштабируемость без индекса | Без пространственного индекса сложность составляет $O(n^2)$, что неприемлемо для больших датасетов |

## Сравнение с другими алгоритмами кластеризации

| Критерий | DBSCAN | K-Means | Mean Shift |
|---|---|---|---|
| Число кластеров | Определяется автоматически | Задаётся заранее | Определяется автоматически |
| Форма кластеров | Произвольная | Сферическая | Произвольная |
| Обработка выбросов | Встроенная | Отсутствует | Частичная |
| Сложность | $O(n \log n)$ с индексом | $O(n \cdot K \cdot T)$ | $O(n^2 \cdot T)$ |
| Кластеры разной плотности | Не поддерживает | Не поддерживает | Частично поддерживает |
| Детерминированность | Для ядровых точек - да | Зависит от инициализации | Да |

---

## Когда использовать

- Кластеры имеют **произвольную форму** (некруглую, вытянутую, вложенную)
- Данные содержат **выбросы**, которые необходимо выявить
- Число кластеров заранее **неизвестно** и не может быть обоснованно задано
- Плотность кластеров **приблизительно одинакова**
- Размерность пространства признаков **невысока** (после применения PCA или иных методов снижения размерности)
- Требуется **интерпретируемый** результат: разделение на кластеры и шум
