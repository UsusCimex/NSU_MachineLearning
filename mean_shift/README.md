# Лабораторная 7 - Mean Shift

## Теоретические основы

**Mean Shift** - непараметрический алгоритм кластеризации, основанный на оценке плотности распределения данных. В отличие от центроидных методов (K-Means), Mean Shift не требует задания числа кластеров заранее - оно определяется автоматически как количество мод (локальных максимумов) плотности.

Алгоритм относится к классу **density-based clustering**: кластеры формируются вокруг областей наибольшей концентрации точек. Каждая точка итеративно сдвигается в направлении возрастания плотности до тех пор, пока не достигнет локального максимума - **моды распределения**. Точки, сошедшиеся к одной моде, образуют один кластер.

### Связь с ядерной оценкой плотности

Mean Shift тесно связан с **Kernel Density Estimation (KDE)** - методом непараметрической оценки функции плотности вероятности. Вектор сдвига среднего указывает в направлении градиента оценки плотности. Таким образом, итеративное применение Mean Shift эквивалентно процедуре градиентного восхождения по поверхности KDE.

### Интуитивный пример

Представим множество точек на плоскости. Если поместить в произвольную точку окно фиксированного радиуса, вычислить среднее положение всех точек внутри окна и переместить центр окна в это среднее - это и есть один шаг Mean Shift. Повторяя процедуру, окно «скатится» к ближайшему скоплению точек - моде плотности.

## Математический аппарат

### Ядерная оценка плотности (KDE)

Для множества точек $x_1, x_2, \ldots, x_n \in \mathbb{R}^d$ ядерная оценка плотности определяется как:

$$\hat{f}(x) = \frac{1}{nh^d}\sum_{i=1}^{n}K\left(\frac{x - x_i}{h}\right)$$

где:
- $n$ - число точек в выборке
- $h$ - ширина окна (bandwidth), управляющая степенью сглаживания
- $d$ - размерность пространства
- $K(\cdot)$ - ядерная функция (kernel)

### Ядерные функции

Наиболее распространённые ядра:

| Ядро | Формула | Область определения |
|------|---------|---------------------|
| Гауссово (Gaussian) | $K(x) = \frac{1}{(2\pi)^{d/2}}\exp\left(-\frac{\|x\|^2}{2}\right)$ | $\mathbb{R}^d$ |
| Плоское (Flat/Uniform) | $K(x) = \begin{cases} 1, & \|x\| \leq 1 \\ 0, & \text{иначе} \end{cases}$ | Единичный шар |
| Епанечникова | $K(x) = \frac{d+2}{2c_d}(1 - \|x\|^2), \quad \|x\| \leq 1$ | Единичный шар |

В `sklearn.cluster.MeanShift` по умолчанию используется **плоское ядро** (flat kernel).

### Вектор сдвига среднего (Mean Shift Vector)

Вектор сдвига среднего для точки $x$ определяется как:

$$m(x) = \frac{\sum_{i=1}^{n} K\left(\frac{x - x_i}{h}\right) x_i}{\sum_{i=1}^{n} K\left(\frac{x - x_i}{h}\right)} - x$$

Числитель дроби - **взвешенное среднее** всех точек выборки с весами, определяемыми ядром. Вектор $m(x)$ всегда направлен в сторону возрастания плотности.

Для плоского ядра формула упрощается до среднего по точкам, попавшим в окно радиуса $h$:

$$m(x) = \frac{1}{|N(x)|}\sum_{x_i \in N(x)} x_i - x$$

где $N(x) = \{x_i : \|x - x_i\| \leq h\}$ - множество точек в окрестности радиуса $h$.

### Алгоритм Mean Shift

1. Для каждой точки $x$ (или подмножества начальных точек - seeds) вычислить вектор сдвига $m(x)$
2. Обновить положение точки: $x \leftarrow x + m(x)$
3. Повторять шаги 1–2 до выполнения критерия сходимости: $\|m(x)\| < \varepsilon$
4. Объединить сошедшиеся точки в кластеры: точки, оказавшиеся в пределах расстояния $h$ друг от друга, относятся к одному кластеру

### Роль параметра bandwidth ($h$)

Параметр $h$ определяет масштаб анализа плотности:

| $h$ | Эффект |
|-----|--------|
| Малое значение | Много мелких кластеров, чувствительность к локальным флуктуациям |
| Большое значение | Мало крупных кластеров, сглаживание структуры данных |
| Оптимальное значение | Число кластеров соответствует реальной структуре данных |

В `sklearn` для автоматической оценки $h$ используется функция `estimate_bandwidth`, основанная на квантилях попарных расстояний между точками:

```python
from sklearn.cluster import estimate_bandwidth
h = estimate_bandwidth(X, quantile=0.3)
```

## Гиперпараметры

Параметры `sklearn.cluster.MeanShift`:

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `bandwidth` | `float` | `None` | Ширина окна (радиус ядра). Если `None`, вычисляется автоматически через `estimate_bandwidth` |
| `seeds` | `array` | `None` | Начальные точки для итераций. Если `None`, используются все точки выборки |
| `bin_seeding` | `bool` | `False` | Использовать дискретизацию пространства для выбора начальных точек. Ускоряет алгоритм на больших данных |
| `min_bin_freq` | `int` | `1` | Минимальное число точек в бине для использования его центра как seed (при `bin_seeding=True`) |
| `cluster_all` | `bool` | `True` | Если `True`, все точки приписываются к ближайшему кластеру. Если `False`, точки-сироты получают метку `-1` |
| `max_iter` | `int` | `300` | Максимальное число итераций для каждой начальной точки |

Параметры `estimate_bandwidth`:

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `quantile` | `float` | `0.3` | Квантиль расстояний, определяющий bandwidth. Увеличение значения увеличивает $h$ |
| `n_samples` | `int` | `None` | Число случайных точек для оценки (ускорение вычислений) |

## Преимущества и недостатки

### Преимущества

| Свойство | Пояснение |
|----------|-----------|
| Не требует задания числа кластеров | Число кластеров определяется автоматически из структуры данных |
| Произвольная форма кластеров | Не предполагает сферическую или иную фиксированную форму |
| Единственный основной гиперпараметр | Только bandwidth ($h$); может быть оценён автоматически |
| Устойчивость к выбросам | При `cluster_all=False` выбросы получают метку `-1` |
| Гарантия сходимости | Алгоритм сходится к стационарным точкам плотности |
| Отсутствие случайной инициализации | Результат детерминирован при фиксированных seeds |

### Недостатки

| Свойство | Пояснение |
|----------|-----------|
| Высокая вычислительная сложность | $O(Tn^2)$, где $T$ - число итераций, $n$ - число точек |
| Чувствительность к bandwidth | Неверный выбор $h$ приводит к избыточному дроблению или слиянию кластеров |
| Плохая масштабируемость | Непрактичен для больших датасетов без `bin_seeding` |
| Проклятие размерности | Эффективность падает в пространствах высокой размерности |
| Предположение о выпуклости мод | Может плохо работать при сильно вытянутых или сложных формах кластеров |

## Сравнение с другими алгоритмами кластеризации

| Критерий | Mean Shift | K-Means | DBSCAN |
|----------|-----------|---------|--------|
| Число кластеров | Определяется автоматически | Задаётся заранее ($K$) | Определяется автоматически |
| Форма кластеров | Произвольная | Сферическая | Произвольная |
| Основной гиперпараметр | bandwidth ($h$) | $K$ | $\varepsilon$, `min_samples` |
| Обработка выбросов | Частичная (`cluster_all=False`) | Нет | Да (метка `-1`) |
| Сложность | $O(Tn^2)$ | $O(TnK)$ | $O(n \log n)$ с деревом |
| Детерминированность | Да | Нет (зависит от инициализации) | Да |

---

## Когда использовать

- Число кластеров заранее неизвестно и нежелательно задавать его вручную
- Кластеры имеют произвольную, не обязательно сферическую форму
- Данные невысокой размерности (2–10 признаков)
- Размер выборки умеренный (до ~10 000 точек без `bin_seeding`)
- Требуется детерминированный результат
- Задача допускает наличие единственного гиперпараметра, оцениваемого автоматически
